## Sebelum anda memulakan

Open API adalah fleksibel dan boleh digunakan dengan atau tanpa token API. Untuk memastikan penggunaan yang bertanggungjawab dan mengelakkan penyalahgunaan, terdapat had pada bilangan permintaan yang pengguna boleh buat.

Bagi pengguna baru, kami mengesyorkan untuk memulakan tanpa pengesahan kerana ia memudahkan proses pembelajaran awal.
Namun, jika anda merasakan anda memerlukan lebih banyak fungsi melebihi had akses tanpa pengesahan, anda mempunyai pilihan untuk [meminta token API](/overview/authentication).

## Menghantar Permintaan

Untuk mengakses OpenAPI, sentiasa gunakan URL asas https://datagovmy.app untuk menghantar permintaan HTTP GET. Apabila membuat permintaan API, tambahkan URL asas ini di hadapan titik akhir API yang ingin anda akses. Berikut adalah contoh permintaan yang mengembalikan maksimum 3 rekod harga bahan api, menggunakan [Data Catalogue API](/static-api/data-catalogue/overview):

```http
GET https://datagovmy.app/catalogue?id=fuelprice&limit=3
```

Dalam contoh ini:

- URL asas ialah `https://datagovmy.app`.
- Titik akhir API tertentu ialah `/catalogue`.
- Parameter kueri yang digunakan ialah `id` dan `limit`.

## Koleksi Postman

Kami mengesyorkan untuk memulakan eksplorasi OpenAPI dengan berinteraksi dengan koleksi Postman kami.
Koleksi tersebut menyediakan persekitaran interaktif di mana anda boleh menghantar permintaan dan menerima respons secara masa nyata.
Ia menawarkan contoh lengkap semua titik akhir yang tersedia, membolehkan anda bereksperimen, menyalin permintaan, dan menyesuaikannya mengikut keperluan anda.
