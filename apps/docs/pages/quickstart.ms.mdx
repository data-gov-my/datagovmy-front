## Sebelum Bermula

OpenAPI ini fleksibel dan boleh digunakan dengan atau tanpa token API. Bagi memastikan penggunaan yang bertanggungjawab dan mengelakkan penyalahgunaan, terdapat had pada bilangan permintaan yang pengguna boleh buat.

Bagi pengguna baru, kami mencadangkan untuk memulakan tanpa pengesahan kerana ia menyederhanakan proses penubuhan awal.
Walau bagaimanapun, jika anda mendapati diri anda memerlukan lebih banyak fungsi melebihi had akses tanpa pengesahan, anda mempunyai pilihan untuk [meminta token API](authentication).

## Menghantar Permintaan

Untuk mengakses OpenAPI, sentiasa gunakan URL asas https://api.data.gov.my untuk menghantar permintaan GET HTTP. Apabila membuat permintaan API, tambahkan URL asas ini ke hujung URL API tertentu yang anda ingin akses. Berikut adalah contoh permintaan yang mengembalikan maksimum 3 rekod harga petrol, menggunakan [API Katalog Data](static-api/data-catalogue):

```http
GET https://api.data.gov.my/data-catalogue?id=fuelprice&limit=3
```

Dalam contoh ini:

- URL asas adalah `https://api.data.gov.my`.
- 'Endpoint' API adalah `/data-catalogue`.
- Parameter pertanyaan yang digunakan adalah `id` dan `limit`.

## Koleksi Postman

Kami mencadangkan anda mulakan penerokaan OpenAPI ini dengan berinteraksi dengan koleksi Postman kami.
Koleksi ini menyediakan persekitaran interaktif di mana anda boleh menghantar permintaan dan menerima respons pada masa nyata.
Ia menawarkan contoh-contoh yang menyeluruh dari semua 'endpoint' yang tersedia, membolehkan anda menguji permintaan, dan menyesuaikannya untuk memenuhi keperluan anda.
